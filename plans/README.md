## Findings considered and rejected

- **image_widget `w*h` overflow** (`image_widget.rs:177`): unreachable —
  `Pixmap::new(w,h)` at line 171 errors first (a >4 GB alloc would have to succeed).
- **applet.rs mutex poisoning in production**: every `lock().unwrap()` is inside
  `#[cfg(test)]`; a poisoned test mutex just fails a test.
- **zipfile path traversal / zip-slip**: `open()` matches an exact entry name and
  returns bytes in memory; nothing is written to disk.
- **multipart upload DoS** (`serve/handlers.rs`): axum's default 2 MB body limit
  bounds each request (no `DefaultBodyLimit` override).
- **`serve` verbose error responses**: by design — localhost-default dev server
  where browser errors help the developer. (`api` is hardened separately in 007.)
- **Two date libraries (jiff + chrono)**: intentional and documented.

## Lower-value items (noted, not planned)

- ~61 clippy *warnings* across render/runtime (do not fail CI). A `cargo clippy
  --fix` pass plus moving CI to `-D warnings` could follow.
- `applet.rs` is ~80% inline tests (~2030 of 2514 lines); moving them to `tests/`
  is cosmetic.
- Per-call regex recompile in `re_module.rs::findall` (low impact).
- O(n²) nearest-colour quantization in `rustlet-encode` (trivial at 64×32).
- Undocumented env vars (`BUILDIFIER`, `RUSTLET_URL`/`PIXLET_URL`,
  `RUSTLET_TOKEN`/`PIXLET_TOKEN`) — a README "Configuration" section would help.
- `module_loader` cycle-detection has no direct test.
